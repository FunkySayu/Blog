Virtual private file-sharing network¶

A while ago, some friends and me were thinking about a system that will allow us to share easily some large files. The point is that in order to work together, we had to share highly weighted files (over 10Go). We are currently hosting each files on a remote server using a FTP network drive.

Context¶

I am working on Linux, Windows and Mac. My friends use Windows and all my servers, of course, are running under Linux. My friends are not IT students and even if they know the basics of computer science, they don't want to spend their time in configuration of a complex system.

The specs¶

The system I was looking have the following specs :

  • it must allow to share huge amount of data
  • all these data are in a file-system form
  • I want a solution that does not overload my server(s)
  • and easy to install (or at least, once installed, work without further manipulations)

The idea¶

The idea is to create a virtual private network between my friends and me, based on one of my servers. All the entities (other servers and end-users owning data), using OpenVPN, connect to this master server and create a Virtual Private Network.

Server Keys generation¶

The OpenVPN end-point must allow multiple users to connect on. Because of this, we need a public key security based server. This implies each user of the VPN must have his own pair of private and public key, used for encryption and authentication. The authentication process is based on certificates ; the administrator generate a valid certificate from the user public key using a Certificate Authority. After this, the end-user will be allowed to join the VPN.

This is a really standard way to create a TLS based VPN. Most of the following pieces of code come directly from the documentation (man openvpn, man openssl).

# Certificate Authority generation
openssl req -new -nodes -x509 -out /etc/ssl/carequest.pem

# DH parameters generation
openssl dhparam 1024 -out /etc/ssl/dh1024.pem

# Local peer keys generation
openssl req -nodes -new -keyout $vpn_keys_dir/private/mycert.key -out $vpn_keys_dir/mycert.csr

# Certificate validation by the certificate authority
openssl ca -out $vpn_keys_dir/mycert.crt -in $vpn_keys_dir/mycert.csr

Caution: be careful while generating your keys. If you are using the default configuration file /etc/ssl/openssl.cnf, the following attributes must have the same value in each certificate :

# For the CA policy
[ policy_match ]
countryName             = match
stateOrProvinceName     = match
organizationName        = match
organizationalUnitName  = optional
commonName              = supplied
emailAddress            = optional

Basic parameters¶

openvpn --dev tun \
        --mode server \
        --tls-server \
        --dh /etc/ssl/dh1024.pem \
        --ca /etc/ssl/cacert.pem \
        --cert $vpn_keys_dir/mycert.crt \
        --key $vpn_keys_dir/private/mycert.key

tun devices encapsulate IPv4 or IPv6 (OSI Layer 3) while tap devices encapsulate Ethernet 802.3 (OSI Layer 2).
